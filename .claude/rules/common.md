# 共通ルール

## コメントとドキュメント

✅ **必須**: すべてのコメントは日本語で記述
```csharp
// ✅ Good
/// <summary>テクスチャを読み込みます</summary>

// ❌ Bad
/// <summary>Load texture</summary>
```

## コーディングスタイル

✅ **DO**:
- 意味のある変数名を使用
- マジックナンバーを定数化
- 1つのファイルは1つの責任

❌ **DON'T**:
- 略語を多用（`tmp`, `str`, `num`）
- 過度にネストしたコード
- コメントアウトされたコード残存

## Gitワークフロー

```bash
# ✅ Good: 明確なコミットメッセージ
git commit -m "シェーダー最適化: テクスチャサンプリング回数を削減"

# ❌ Bad: 不明確なメッセージ
git commit -m "fix"
```

## セキュリティ

✅ **必須**:
- `.env` ファイルは `.gitignore` に追加
- APIキーやパスワードをハードコードしない
- 機密情報は環境変数で管理

## パフォーマンス

- 早すぎる最適化を避ける
- まずProfilerで計測
- ボトルネックを特定してから最適化
